## 🔍 根本原因分析

### 问题 1: 验证码验证机制导致测试失败

**现象**: 注册和登录接口返回 401，提示"验证码错误或已过期"

**原因**: 
- 测试中使用的验证码 `'123456'` 是硬编码的
- 实际验证码存储在 `verificationCodeMap` 中，需要先调用 `send-code` 生成
- 验证码有 5 分钟过期时间，且是一次性的

**解决方案**:
```javascript
// 修改测试流程，先获取真实验证码
beforeEach(async () => {
  // 1. 发送验证码
  await request(app).post('/api/auth/send-code').send({ phone: TEST_PHONE });
  
  // 2. 从内存中获取验证码 (测试环境需要暴露此接口)
  // 或者使用测试专用的 bypass 验证码
});
```

### 问题 2: 健康检查路由 404

**现象**: GET /health 返回 404

**原因**: 健康检查路由可能未正确挂载

**解决方案**: 检查 `src/server.js` 中是否包含:
```javascript
app.use('/health', healthRoutes);
```

### 问题 3: 端口占用警告

**现象**: 测试中出现 `EADDRINUSE address already in use :::3000`

**原因**: 多个测试同时启动服务器实例

**解决方案**: 使用 `beforeAll` 启动一次服务器，`afterAll` 关闭
