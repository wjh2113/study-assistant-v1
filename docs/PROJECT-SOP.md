## 1. 团队角色与职责

### 1.1 项目经理（PM）

**核心职责**：统筹协调、任务分配、进度跟踪、决策汇报

| 工作项 | 频率 | 输入 | 输出 |
|--------|------|------|------|
| 项目计划制定 | 项目启动 | 俊哥需求、PRD | 项目计划表 |
| 任务分配 | 每阶段开始 | PRD、架构方案 | 任务清单 |
| 进度跟踪 | 持续 | 各团队进度 | 进度看板 |
| 进度汇报 | **每 30 分钟** | 各团队进度 | 俊哥汇报 |
| 决策记录 | 按需 | 风险升级 | 决策记录 |
| 最终验收 | 项目结束 | 交付物 | 验收报告 |

**关键技能**：
- 项目管理（PMP/敏捷）
- 沟通协调
- 风险管理
- 决策能力

### 1.2 产品设计师（Product-UX）

**核心职责**：需求分析、产品设计、原型设计、PRD 输出

| 工作项 | 频率 | 输入 | 输出 |
|--------|------|------|------|
| 需求分析 | 项目启动 | 俊哥需求 | 需求清单 |
| 功能设计 | 设计阶段 | 需求清单 | 功能清单 |
| 用户流程设计 | 设计阶段 | 功能清单 | 用户流程图 |
| 原型设计 | 设计阶段 | 功能清单 | 原型（Figma/墨刀） |
| PRD 编写 | 设计阶段 | 以上所有 | PRD 文档 |
| 设计评审 | 评审会议 | 架构师意见 | PRD 修订版 |

**关键技能**：
- 产品设计
- 用户体验设计
- 原型工具（Figma/墨刀）
- 文档编写

**输出文档**：
| 文档 | 说明 | 完成标准 |
|------|------|----------|
| `PRD_vX.X.md` | 产品需求文档（主文档） | 功能清单完整、优先级明确 |
| `user-flow.md` | 用户流程图 | 学生/家长核心流程清晰 |
| `prototype.md` | 原型设计（Figma/墨刀链接） | 高保真原型 + 交互说明 |
| `feature-list.md` | 功能清单（P0/P1/P2 优先级） | 与 PRD 一致 |

### 1.3 架构师（Architect）

**核心职责**：6 大架构设计 + 评审 + 风险识别

| 架构维度 | 设计内容 | 输出文档 |
|----------|----------|----------|
| **业务架构** | 用户流程、业务边界、价值流 | 业务流程图 |
| **应用架构** | 模块拆分、接口规范、部署拓扑 | 架构拓扑图 |
| **数据架构** | ER 图、数据流、存储策略、备份方案 | 数据架构图 |
| **技术架构** | 技术选型、性能指标、扩展方案 | 技术选型报告 |
| **安全架构** | 认证授权、数据加密、审计日志、合规 | 安全评审报告 |
| **AI 架构** | 模型路由、Prompt 工程、成本控制、降级策略 | AI 架构方案 |

| 工作项 | 频率 | 输入 | 输出 |
|--------|------|------|------|
| PRD 评审 | 设计阶段结束 | PRD 文档 | 评审意见 |
| 6 大架构设计 | 设计阶段 | PRD+ 评审意见 | 架构文档 |
| 风险识别 | 持续 | 架构设计 | 风险清单 |
| 架构决策 | 按需 | 技术方案 | ADR 文档 |
| 补充 PRD | 设计阶段结束 | 架构文档 | PRD 架构章节 |

**关键技能**：
- 系统架构设计
- 技术选型
- 安全架构
- 风险管理

**输出文档**：
| 文档 | 说明 | 完成标准 |
|------|------|----------|
| `architecture-review-report.md` | 架构评审报告 | 6 大维度评分 + 风险清单 |
| `architecture-risks.md` | 架构风险清单 | P0/P1/P2 优先级 + 修复方案 |
| `adr-XXX.md` | 架构决策记录（每决策一个） | 问题/方案/决策/后果 |
| `PRD_vX.X.md` 架构章节 | 补充到 PRD 的技术选型/部署设计 | 与 PRD 整合 |
