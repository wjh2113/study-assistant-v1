## 2. 开发优先级建议

### 2.1 推荐开发顺序

基于**用户价值**和**技术依赖**两个维度，建议按以下顺序开发：

| 优先级 | 功能模块 | 预计工时 | 理由 |
|--------|----------|----------|------|
| **1** | 个人中心页面 | 0.5 天 | 独立功能，用户刚需，为其他功能提供入口 |
| **2** | 课本上传模块 | 2 天 | 核心数据源，无依赖，后续功能基础 |
| **3** | 智能练习模块 | 3 天 | 核心学习闭环，依赖课本模块 |
| **4** | 积分系统前端 | 1 天 | 依赖练习模块，激励体系核心 |
| **5** | 排行榜前端 | 1 天 | 依赖积分系统，社交激励 |
| **6** | 家庭绑定流程 | 1.5 天 | 独立流程，家长端核心功能 |
| **7** | 薄弱点分析前端 | 1.5 天 | 依赖练习数据积累，可 V1.1 实现 |

**总预计工时**: 10.5 天（按单人全职计算）

### 2.2 分阶段交付建议

#### 阶段一：基础闭环（4.5 天）
- 个人中心页面
- 课本上传模块
- 智能练习模块（基础版）

**目标**: 学生可上传课本并完成练习

#### 阶段二：激励体系（2 天）
- 积分系统前端
- 排行榜前端

**目标**: 形成学习 - 积分 - 排名闭环

#### 阶段三：家庭联动（3 天）
- 家庭绑定流程
- 薄弱点分析前端（可选）

**目标**: 家长可参与学生学习过程

| progress | number | 系统生成 | 解析进度 0-100 |

| updatedAt | timestamp | 系统生成 | 更新时间 |

### 6.4 API 接口对接建议

| 页面 | 需要调用的 API | 说明 |
|------|---------------|------|
| 课本上传 | `POST /api/textbooks` | 创建课本记录，返回 uploadUrl |
| 课本列表 | `GET /api/textbooks` | 获取当前用户课本列表 |
| 课本详情 | `GET /api/textbooks/:id` | 获取课本详情 + 单元树 |
| 创建练习 | `POST /api/practice/sessions` | 创建练习会话，返回题目 |
| 提交答案 | `POST /api/practice/sessions/:id/answers` | 提交单题答案 |
| 练习结算 | `POST /api/practice/sessions/:id/finish` | 完成练习，获取积分 |
| 积分明细 | `GET /api/points/ledger` | 获取积分流水 |
| 排行榜 | `GET /api/leaderboard` | 获取排名列表 |
| 家庭绑定 | `POST /api/family/bind-requests` | 发起绑定请求 |
| 绑定确认 | `POST /api/family/bind-requests/:id/confirm` | 确认绑定 |
