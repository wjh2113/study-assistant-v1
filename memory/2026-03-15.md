## 👥 团队结构 (6 人)

1. **项目经理 (PM)** - 统筹协调、任务分配、进度跟踪、决策汇报 (每 30 分钟向俊哥汇报)
2. **产品设计师 (Product-UX)** - 需求分析、产品设计、原型设计、PRD 输出
3. **架构师 (Architect)** - 6 大架构设计 (业务/应用/数据/技术/安全/AI) + 评审 + 风险识别
4. **项目助理** - 文档审查、过程审计、GitHub 同步 (每 30 分钟)、质量把控、变更管理、文档归档
5. **fullstack** - 后端/前端开发、数据库设计、单元测试
6. **qa** - API 测试、前端测试、浏览器 E2E、测试报告、Bug 跟踪

---

## 📊 工作流程 (4 阶段)

1. **阶段 1: 需求分析与设计** (1-2 周)
   - 产品设计师 → 架构师 → 项目经理
   - 输出：PRD+ 架构文档

2. **阶段 2: 开发实施** (2-4 周)
   - fullstack → 项目助理 (代码审查+GitHub 同步)
   - 输出：源代码+API 文档

3. **阶段 3: 测试验证** (1-2 周)
   - qa → fullstack(Bug 修复) → 项目助理
   - 输出：测试报告+Bug 清单

4. **阶段 4: 交付归档** (3-5 天)
   - 项目助理 → 项目经理 → 俊哥
   - 输出：交付文档+GitHub
