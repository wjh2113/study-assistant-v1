## 🔴 阻塞性 Bug (P0 - 立即修复)

### BUG-P0-001: Redis 版本不兼容导致队列功能失效

**优先级**: P0 - 阻塞性  
**状态**: ⏳ 待修复  
**影响范围**: 课本解析 Worker、排行榜计算、异步任务队列  
**发现时间**: 2026-03-17 06:43 (最新测试中发现)

#### 问题描述
系统安装的 Redis 版本为 3.0.504，但 BullMQ 队列库要求 Redis 版本 > 5.0.0。这导致：
- 队列功能完全不可用
- 课本解析 Worker 无法正常工作
- 排行榜定时计算任务无法执行
- 所有异步任务阻塞

#### 错误信息
```
Error: Redis version needs to be greater than 5.0.0 Current: 3.0.504
    at RedisConnection.init (bullmq/src/classes/redis-connection.ts:189:15)
Code: ERR_UNHANDLED_ERROR
```

#### 复现步骤
1. 启动后端服务
2. 查看控制台日志
3. 出现 Redis 版本错误
4. 队列相关功能全部失败

#### 修复方案

**方案 A: 升级 Redis (推荐)**
```bash
# Windows 环境
# 1. 下载 Redis 5.0+ (或使用 WSL2/Docker)
docker pull redis:7.0
docker run -d -p 6379:6379 redis:7.0

# 2. 更新 docker-compose.yml
services:
  redis:
    image: redis:7.0
    ports:
      - "6379:6379"
```

**方案 B: 降级 BullMQ 版本 (临时方案)**
```bash
# 安装支持 Redis 3.x 的旧版 BullMQ
npm install bullmq@1.90.0 --save

# 注意：会丢失新特性，仅作为临时方案
```

**方案 C: 禁用队列功能 (开发环境)**
```javascript
// backend/src/config/queue.js
if (process.env.NODE_ENV === 'development') {
  // 使用内存队列替代
  return { useMemoryQueue: true };
}
```

#### 验证步骤
- [ ] Redis 版本检查：`redis-server --version` 应显示 > 5.0.0
- [ ] 队列连接测试：启动后端无错误日志

- [ ] 排行榜测试：定时任务正常运行

## 🟠 严重 Bug (P1 - 高优先级)

### BUG-P1-001: 用户注册邮箱格式验证缺失

**优先级**: P1 - 严重  
**状态**: ⏳ 待修复  
**影响范围**: 用户注册模块  
**发现时间**: 2026-03-14 (TEST_REPORT_FINAL.md)  
**报告来源**: QA Sub-Agent 深度测试

#### 问题描述
注册接口未验证邮箱格式，允许无效邮箱（如 `invalid-email`）注册成功，可能导致：
- 用户无法接收验证码
- 垃圾账号注册
- 数据质量下降

#### 错误信息
```
测试用例：无效邮箱格式注册
预期结果：返回 400 错误
实际结果：返回 201 注册成功 ❌
```

#### 复现步骤
1. 访问注册页面
2. 输入用户名：`testuser`
3. 输入邮箱：`invalid-email` (无@符号)
4. 输入手机号：`13800138000`
5. 提交注册
6. 结果：注册成功 (不应成功)

#### 修复方案
```javascript
// backend/src/controllers/authController.js
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;

async function register(req, res) {
  const { email } = req.body;
  
  if (!email || !emailRegex.test(email)) {
    return res.status(400).json({ 
      error: '邮箱格式无效',
      code: 'INVALID_EMAIL_FORMAT'
    });
  }
  
  // ... 继续注册流程
}
```

#### 验证步骤
- [ ] 输入 `invalid-email` → 返回 400
- [ ] 输入 `test@example.com` → 允许继续
- [ ] 输入空邮箱 → 返回 400
- [ ] 前端同步添加验证提示

#### 预计修复时间**: 15 分钟

### BUG-P1-002: XSS 安全风险 - 特殊字符未转义

**优先级**: P1 - 严重  
**状态**: ⏳ 待修复  
**影响范围**: 知识点管理、所有用户输入内容展示  
**发现时间**: 2026-03-14 (TEST_REPORT_FINAL.md)  
**报告来源**: QA Sub-Agent 安全测试

#### 问题描述
创建知识点时，标题和内容中的特殊字符（如 `<script>` 标签）未被转义或过滤，可能导致 XSS 攻击。

#### 错误信息
```
测试用例：XSS 注入尝试
输入：<script>alert('xss')</script>
预期：转义或拒绝
实际：原样存储并返回 ⚠️
```

#### 复现步骤
1. 创建知识点
2. 标题输入：`<script>alert('XSS')</script>`
3. 查询知识点列表
4. 结果：返回未转义的内容，浏览器可能执行脚本

#### 修复方案

**方案 A: 存储时转义 (推荐)**
```javascript
// backend/src/controllers/knowledgeController.js
const escape = require('escape-html');

async function createKnowledge(req, res) {
  const { title, content } = req.body;
  
  const knowledge = await prisma.knowledgePoint.create({
    data: {
      title: escape(title),
      content: escape(content),
      // ...
    }
  });
}
```

**方案 B: 输出时转义**
```javascript
// 前端展示时转义
// frontend/src/pages/Knowledge.jsx
const escapeHtml = (text) => {
  const map = {
    '&': '&amp;',
    '<': '&lt;',
    '>': '&gt;',
    '"': '&quot;',
    "'": '&#039;'
  };
  return text.replace(/[&<>"']/g, m => map[m]);
};
```

#### 验证步骤
- [ ] 创建含 `<script>` 的知识点 → 存储为转义后内容
- [ ] 前端展示 → 显示为文本而非执行脚本
- [ ] API 返回 → 内容已转义

#### 预计修复时间**: 30 分钟
