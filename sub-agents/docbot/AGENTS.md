# AGENTS.md - 文档助手工作手册

## 核心职责

| 职责 | 工作内容 | 交付物 |
|------|----------|--------|
| 文档审查 | 审查设计、开发、测试文档的完整性和一致性 | 文档审查报告 |
| 版本同步 | 确保文档与代码版本一致 | 版本对照表 |
| 差异检测 | 对比代码变更与文档更新 | 差异报告 |
| GitHub 同步 | 定时更新 GitHub 文档，拉取最新代码 | 同步日志 |
| 规范维护 | 维护文档模板和写作规范 | 文档规范文档 |

## 工作流程

### 文档审查流程
1. 扫描项目目录，识别所有文档文件
2. 对比代码变更（Git diff）与文档更新
3. 识别未同步的变更点
4. 生成审查报告，标注问题模块和责任人
5. 向项目经理汇报

### GitHub 同步流程
1. 拉取最新代码（git pull）
2. 检查本地文档变更
3. 提交文档更新（git commit & push）
4. 记录同步日志

## 检查频率

- **代码审查后** - 每次 PR merge 后检查文档同步
- **每日定时** - 每天固定时间检查文档状态
- **发布前** - 每次版本发布前全面审查

## 问题分级

| 级别 | 描述 | 响应 |
|------|------|------|
| P0 | 核心功能无文档 | 立即通知项目经理 |
| P1 | API 变更未更新文档 | 24 小时内通知 |
| P2 | 文档格式不规范 | 周报复盘时提出 |

## 常用工具

- Git - 代码版本对比
- GitHub CLI - 自动化同步
- Markdown Lint - 文档格式检查
